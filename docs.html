Markfolio Docs

A fast, zero-install markdown editor that runs entirely in your browser. No accounts, no servers — just write.

Getting Started

Open the editor

Visit the app in any modern browser — no install, no build step, no server needed. Your notes are saved automatically to localStorage and persist across sessions.

https://akshaykokane.github.io/markfolio/index.html

Interface Overview

Three-pane layout

  • Sidebar (left) — list of all your notes. Click a note to switch to it.
  • Editor (center) — write raw Markdown here.
  • Preview (right) — live-rendered HTML output, updated as you type.

Toolbar

  • Title field — the note title. Used as the filename when exporting.
  • Export button — downloads the current note as a .zip containing the .md file and any pasted images.
  • Docs button — opens this page.

Managing Notes

Create a note

Click the + button in the sidebar header to create a new blank note.

Switch notes

Click any note in the sidebar. The current note is saved automatically before switching.

Delete a note

Hover over a note in the sidebar to reveal the × delete button. If only one note exists, deleting it clears its content rather than removing it — there is always at least one note.

Images

Paste an image

Copy an image to your clipboard, click inside the editor, and press Ctrl+V / Cmd+V. The image is stored in memory and inserted as a markdown image tag automatically.

Drag and drop

Drag an image file from your file system and drop it onto the editor area. It will be inserted at the cursor position.

Images persist across refreshes

Images are saved to IndexedDB in your browser, keyed per note. They are restored automatically when you reload the page or switch back to a note. Deleting a note also removes its images from IndexedDB.

Exporting

Export as ZIP

Click ⬇ Export in the toolbar. A .zip file is downloaded containing:

  • {title}.md — the markdown source at the root
  • assets/images/ — all pasted images from the current session

The structure is ready to commit directly to a GitHub Pages or other static site repository.

Data & Privacy

Note text is stored in localStorage under markfolio_notes. Images are stored in IndexedDB under the database markfolio_images. Nothing is sent to any server. Clearing browser storage or site data will erase both notes and images, so export important work regularly.
